# Definition Of Done

This file defines what counts as complete for major project phases. Do not mark a phase done based on partial progress.

## 1. Business Model Done

This phase is complete when:

- the first niche is chosen
- the core offer is defined
- Launch, Growth, and Premium tiers are specified
- standard vs custom work is clearly separated
- recurring offers are defined
- human approval gates are documented

## 2. Starter Architecture Done

This phase is complete when:

- the Next.js base app exists
- Payload CMS is wired in
- base SEO setup exists
- forms and analytics baseline exists
- design tokens exist
- core reusable blocks exist
- at least one niche-oriented template family exists
- deployment path is defined
- lead-capture success states represent real persistence or delivery, not placeholders
- public routes do not expose draft CMS content through direct slug access
- the baseline validation commands actually run in the repo as configured

## 3. Sales Workflow Done

This phase is complete when:

- lead schema exists
- qualification logic exists
- outreach draft workflow exists
- discovery brief structure exists
- proposal template exists
- CRM stage model exists
- human review points are defined

## 4. Delivery Workflow Done

This phase is complete when:

- project bootstrap flow exists
- content planning flow exists
- design planning flow exists
- QA checklist exists
- launch checklist exists
- handoff and support process exists
- feature-gated behavior is verified for both enabled and disabled paths where relevant

## 5. Agent Workflow Done

Any single agent workflow is complete only when:

- the task is clearly scoped
- inputs are defined
- outputs are defined
- prompt or instructions are versioned
- success criteria are defined
- failure handling is defined
- required human review is defined

## 6. Outreach MVP Done

This phase is complete when:

- there is a lead sourcing method
- lead enrichment fields are defined
- outreach templates exist
- outreach review process exists
- sends can be tracked
- reply and meeting metrics can be recorded

## 7. Project Ready For Real Clients

This phase is complete when:

- one niche and one template family are operational
- service tiers are publishable
- discovery and proposal flow can run end to end
- starter stack can generate a delivery-ready project
- QA and launch review process exists
- a basic agency site and demo assets exist
- lead capture stores or delivers real submissions safely
- lint, typecheck, and critical e2e flows pass without manual config repair
