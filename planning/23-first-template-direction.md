# First Template Direction

This file defines the next product-facing planning step after CI hardening, deployment documentation, and schema cleanup.

The goal is to make the first home-services demo and template-family direction concrete enough that future implementation work does not drift into generic starter work again.

## Why This Is The Next Step

The repo now has:

1. a working Next.js + Payload starter
2. validated core schema and content boundaries
3. documented deployment and CI baselines

The next missing product asset is not more infrastructure.

The next missing product asset is a credible first template direction and demo brief that supports outreach, sales conversations, and repeated delivery.

## Active Recommendation

Define one strong home-services template family before building multiple niche variants.

Recommended direction:

1. one template family for owner-led or small-team GTA home-service businesses
2. optimized for credibility, conversion, and local SEO rather than heavy brand expression
3. flexible enough to fit common categories like plumbing, HVAC, electrical, roofing, and landscaping without changing core page architecture

## Template Family Intent

The first template family should feel:

1. trustworthy
2. fast to scan
3. lead-oriented
4. strong on mobile
5. local-service credible

It should emphasize:

1. primary CTA clarity
2. trust blocks near the top of the page
3. service coverage and local area proof
4. review and credential visibility
5. straightforward service-page depth

It should avoid:

1. over-designed brand experiments
2. niche-specific logic that only fits one trade
3. bespoke page architecture before real sales feedback exists

## Demo Site Brief

Build the first demo as a believable Growth-tier home-services site.

Recommended demo profile:

1. single-location GTA operator
2. multiple core services
3. strong local lead-generation focus
4. enough page depth to demonstrate the reusable system

Recommended demo deliverables:

1. homepage
2. services index
3. at least 3 individual service pages
4. service-areas index if enabled by tier
5. at least 2 service-area pages
6. contact page
7. about or company page if needed for trust and sales positioning

## Content Priorities For The Demo

The first demo should prove these things well:

1. headline and CTA structure
2. trust indicators and social proof
3. service breakdown clarity
4. local SEO structure
5. conversion-ready contact flow

Nice-to-have later:

1. blog content depth
2. richer premium-only blocks
3. advanced estimator or booking behavior

## Out Of Scope For This Step

1. final pricing decisions
2. CRM selection
3. multi-template expansion
4. second niche work
5. advanced automation
6. custom app-like interactions

## Done Condition

This step is complete when:

1. the first template family direction is explicitly chosen
2. the first demo site brief is concrete enough to implement without improvising the business intent
3. the chosen demo scope aligns with Launch/Growth/Premium boundaries already documented elsewhere
4. future implementation work can point to one product-facing target instead of generic starter polish

## Recommended Follow-Up After This

1. lock the demo business profile and site map
2. define the content and block plan for that demo
3. implement the demo pages without reopening infrastructure work unless a real blocker appears
