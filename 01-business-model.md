## Tier Fit Rules

Use these rules when mapping leads or projects to a package:

1. Launch if the client mainly needs credibility, clear contact paths, and a fast replacement for a weak site.
2. Growth if the client needs lead generation, service-page depth, service-area coverage, or stronger conversion structure.
3. Premium if the client has more complex content, higher brand expectations, or approved custom needs that still fit the agency model.

## Working Pricing Logic

Pricing should be based on package tier plus bounded add-ons, not open-ended custom quoting.

Use:

1. a base package price by tier
2. clearly priced add-ons for extra page groups, extra integrations, premium content structure, or custom features
3. explicit exclusions for anything outside the package model

Do not finalize exact prices until the starter architecture and delivery time assumptions are clearer.

## Scope Rules

Standardized:

- base architecture
- block library
- design tokens
- page templates
- CMS structure
- standard integrations
- QA process

Custom / approval required:

- unusual business logic
- advanced integrations
- custom app-like workflows
- deep brand redesigns
- ecommerce-heavy features

## Margin Logic

Profitability comes from:

1. niche focus
2. standardized build system
3. repeatable sales discovery
4. tiered scope control
5. retainers after launch

It does not come from trying to fully automate chaotic custom work.

## Decisions Already Locked

1. Start narrow instead of broad.
2. Use productized service tiers.
3. Keep human review in high-trust moments.
4. Use Next.js + Payload CMS.

6. Start with home services as the first niche.
